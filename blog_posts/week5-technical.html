What are Ruby Classes?

Simply put, Classes are a Ruby construct that allows us to implement properties and inherit those properties by its instantiated objects. At the heart of Ruby, we have a lot of built-in classes that provide us with the initial set of tools that act as a starting point for the creation of our own classes and objects. In Ruby, everything is an object--including Classes--which means a typical class consists of a collection of method definitions, or properties, belonging to that particular class. When we create an instance of a class, that newly created object will inherit those properties the parent class contained.

For our example, I’m creating a video game and I need a construct for the bad guys so I can create a bunch of them without having to code each one. After all, I already know they will all be similar; each will have strength, health, and I’m going to define another method that gives me his status. So, it’s a perfect time to use classes!

	class Badguy

		def initialize(strength, health)
			@strength = strength
			@health = health
		end

		def status(@strength, @health)
			"My strength is #{@strength} and my health is at #{@health}!”
		end
	end

We then create an instance of a class using the built-in .new method:

	jake = Badguy.new(10, 100)
	#=> this jake object has a strength of 10 and health of 100 and belongs to the Badguy class.

That’s it! We now have a new bad guy named jake. Don’t believe me? Well, let's check!

	jake.is_a?(Badguy)
	#=> True

Now, let’s try to get jake to do his thing and tell us his status. We simply call on the method that the class he was instantiated from includes.

	jake.status
	#=> “My strength is 10 and my health is at 100!”

Jake’s status method is actually called an instance method. That is because the method status was defined within the Badguy class and can only be used by objects instantiated by that class construct. If I tried to call this method on an object that was not apart of the Badguy class, I will receive a “method not defined” error.

You might have also noticed a method I defined called initialize right from the beginning of the class with an assignment of the argument to a variable with a @ infront of it. Variables with @ in-front of them mean they are instance variables. Ruby objects come with their own storage and retrieval mechanism for values which enables individual objects to remember state. These instance variables are only visible to the object to which they belong. Also, instance variables initialized in one method inside a class can be used by any method defined within that class. That’s how we got to pass those instance variables created in one method of the class, initialize, into the other status method which then used that information to run it’s own tasks.

The beauty of Ruby classes are flexible and versatile. For example, we can easily close and reopen any class at any time to alter its properties. I could open the Badguy class again, in a different location, and add or overwrite its methods.

Another cool feature is the ability to make the methods within the class private. By simply adding the special keyword private to our class makes ALL METHODS BELOW it un-callable from outside of the class. So for example, if I put private above the status method in our previous example, I would be unable to call that status method on any objects, even if they were instantiated from that class.

Classes are a powerful construct to model objects and share their properties and data by allowing its instances to inherit their properties. This is an excellent way to create a batch of objects that all share a common ancestry and have similar capabilities. Read more about inheritance with Ruby Modules.
